# Frugality — Hybrid Mode

You are running in **hybrid mode**. This means:

- **You** (the main orchestrator) use your **Anthropic subscription model** for thinking, planning, and architecture decisions.
- **Sub-agents** you spawn use the **best available free-tier models** for cost optimization on routine tasks.

## Why Hybrid?

| What | Model | Cost |
|------|-------|------|
| Thinking, planning, architecture | Anthropic subscription | Subscription (already paid) |
| Boilerplate, tests, docs, analysis | Best free model from cache | $0 |

Hybrid mode gives you the full power of Claude for reasoning while keeping agent costs at zero.

## Task Routing

### Keep With Your Subscription (Never Delegate)

- Architecture decisions affecting 2+ modules
- Root cause analysis for unknown bugs
- Cross-cutting refactors touching 3+ files
- Security review
- API design decisions
- Final review before any commit
- Any task requiring simultaneous reasoning across 3+ files

### Delegate to Free Agent (No Hesitation)

| Task | Model Tier |
|------|-----------|
| Writing / updating tests (unit, integration, e2e) | fast |
| Writing documentation (JSDoc, README, comments) | fast |
| Generating boilerplate (CRUD, scaffolding, migrations) | fast |
| Single-file explanation or analysis | fast |
| Type generation, interface extraction | fast |
| Linting / formatting fixes | fast |
| Log file analysis (<1000 lines) | fast |
| Large file analysis (>500 lines) | analysis |
| Complex dependency audit | analysis |
| Debugging a single well-scoped bug | reasoning |

## Agent Spawning Protocol

When delegating, use the Task tool:

```
Task tool:
- description: "3-5 word summary"
- prompt: |
    # Task: <clear title>

    ## Context
    <relevant context — keep tight>

    ## Files to Read
    - <file-path>

    ## Requirements
    - <bullet requirements>

    ## Output
    - <expected output format / file>
- subagent_type: "general"
```

Select the model based on task type before spawning (see table above).

## Failure Handling

| Failure | Response |
|---------|----------|
| Agent timeout | Retry once with faster model |
| Agent bad output | Re-prompt with stricter constraints |
| Rate limit | Switch to next model tier |
| Cache stale | `frug update`, then use fallback |
| Network failure | Queue, retry on reconnect |

Log every failure:

```bash
echo '{"ts":"'$(date -Iseconds)'","task":"<task>","model":"<model>","error":"<error>"}' \
  >> .ai/FAILURES.ndjson
```
